# Part B Review Brief — deepreadqa (online AgenticRAG progressive-reading QA)

## Goal
Answer CAE-eval questions by an agentic loop that uses the DeepRead store (Part A, approved) via progressive-reading tools, then composes a rubric-aligned answer. Output `{item_idx, answer}` JSONL for cae-rubrics-eval. Target: match/exceed agenticRAG's 0.823 mean_anchored. The online opus-4.8 tool-calling on aiberm has been VALIDATED working (real 2-item smoke: search→head→grep, natural stop, no errors).

## Module contracts (what to verify)
- `config.py`: frozen `Endpoint(name, base_url, api_key, model, omit_temperature)` + frozen `Config` (loop/budget/retrieval/compose params). `Config.from_env()` reads `AIBERM_BASE_URL`/`AIBERM_API_KEY` (credentials) + `DEEPREAD_AGENT_MODEL` (model selection — by design DEEPREAD_*, not AIBERM_*); sets `omit_temperature=True` (opus on aiberm rejects temperature). Key params: max_iterations=15, token_threshold=128000, max_output_tokens=2000, results_per_query=8, grep_token_cap=9000, raw_token_cap=40000, concise_compose=True, compose_evidence_token_cap=40000, compose_max_tokens=1300.
- `llm.py`: single-endpoint `ToolLLM(endpoint, *, request_timeout_s, max_retries_per_endpoint)`; `chat(messages, *, tools=None, tool_choice="auto", max_tokens=None) -> LLMResponse(content, tool_calls, finish_reason, total_tokens, raw_message)`. Never sends temperature when omit_temperature; narrows temperature-complaint detection to require an unsupported/not-support/deprecat qualifier; retries with backoff (no sleep after final attempt); `LLMError` after exhaustion. Token accounting by design: `LLMResponse.total_tokens` = THIS call's usage; the `ToolLLM.total_tokens` INSTANCE attribute accumulates across calls and is what `AgentResult.total_tokens` reports (the harness resets it per question). tool_calls returned as raw OpenAI objects (.id/.function.name/.function.arguments).
- `tokens.py`: `count_messages_tokens(messages)` reuses `deepread_sdk.tokens.count_tokens`; handles str + list content + per-message overhead.
- `retrieval.py`: `tokenize_mixed` (regex latin/digit + jieba CJK); `SearchHit(doc_id,title,tldr,score,section_name,section_idx)`; `SearchIndex(reader)` BM25 over doc-summary + section units; `search` aggregates to doc via MAX and surfaces the best-matching SECTION as hint; `search_many` dedups across queries keeping max, sorted desc. Empty-index guard.
- `prompts.py`: `SYSTEM_PROMPT` (progressive-reading discipline: bilingual queries → search → head → intro/preview/read_section → grep → enough? → raw verify → answer-first with doc_id/section_name citations), `FORCE_FINAL_PROMPT`, `FORCE_SUMMARIZE_PROMPT`, `COMPOSE_SYSTEM` (rubric-aligned: answer-first, decisive, concise, no abstain, cite), `COMPOSE_USER_TEMPLATE` (placeholders {question},{evidence},{draft_block}).
- `tools.py`: `TOOL_SCHEMAS` (8: search/head/read_section/intro/preview/grep/read_raw/summarize); `ToolBox(cfg, reader, index)` with `execute(name,args)->str` + `seen_docs`. Budget-controlled compact views; grep O(n) token cap; read_raw cap; graceful unknown-doc (KeyError→"not found") and unknown-tool handling; seen_docs tracked by every doc-touching tool.
- `harness.py`: `AgentResult(answer, full_answer, iterations, total_tokens, compactions, forced_final, error, tool_calls, seen_docs)`; `DeepreadQA(cfg, *, llm=None, reader=None, index=None).answer(question)`. Loop ≤ max_iterations; over token_threshold → force-compress; no tool_calls → finalize (concise compose if enabled) → return (forced_final=False); else execute tools, append OpenAI-shaped messages; model summarize → prune (compactions++); LLMError/exhaustion → forced finish (forced_final=True). `_prune` keeps system + first user message + summary. `_finalize` runs the compose head over evidence (bounded by compose_evidence_token_cap) when concise_compose.
- `run_eval.py`: loads CAE-eval.json, runs the agent per item, writes scorer `{item_idx, answer}` JSONL + rich telemetry; per-item crash isolation (answer=""); flush per item; --output/--ids/--limit/--shard/--num-shards/--no-concise; validates --num-shards>0.
